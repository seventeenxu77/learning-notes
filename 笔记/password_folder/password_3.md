#### 第三次课后作业
##### 作业：完成对3-DES的Merkle-Hellman攻击：令A=0，再对K1的所有$2^{56}$个取值，找出令A=0的明文。补全该算法，以描述或伪代码方式
---


##### 密钥 $K_1$ 的筛选与 $A=0$ 数据对的获取

找到所有 $2^{56}$ 个可能的 $K_1$ 取值，并为每个 $K_1$ 生成一个特殊的明文-密文对 $(P^*, C^*)$，该明文 $P^*$ 使得第一个加密中间值 $A$ 满足 $A=0$。


1. 遍历 $K_1$ 的所有 $2^{56}$ 个可能取值 $i$。对于每一个 $i$，利用 DES 加密的可逆性来计算出满足 $E_{i}(P^*) = \mathbf{0}$ 的明文 $P^*$。 $P^*$ 可以唯一确定为 $P^* = D_i(\mathbf{0})$。
2. 通过执行选择明文攻击，将计算出的明文 $P^*$ 输入到 3DES 目标系统中，获取对应的最终密文 $C^*$。
3. 将所有 $2^{56}$ 个可能的密钥 $K_1=i$ 及其对应的密文 $C^*$ 存储在一个列表 $\text{List}_{\text{Forward}}$ 中。


##### 反向计算与哈希表构建

利用任一参考密文 $C^*$，计算所有可能的 $K_3$ 对应的反向中间值 $B_{\text{rev}}$.

1. 从 $\text{List}_{\text{Forward}}$ 中选取任意一个密文 $C^*_{\text{ref}}$ 作为参考,遍历 $K_3$ 的所有 $2^{56}$ 个可能取值 $k$。
2. 对于每个 $k$，计算反向中间值 $B_{\text{rev}} = D_{k}(C^*_{\text{ref}})$。
3. 构建一个哈希表 $\mathcal{H}_{\text{Backward}}$，将计算得到的 $B_{\text{rev}}$ 作为键，对应的 $K_3=k$ 作为值存储起来。

##### 中间相遇与密钥验证

遍历所有 $K_2$ 和 $K_1$ 的组合，计算正向中间值 $B_{\text{fwd}}$，并在哈希表中寻找匹配的 $K_3$。

1. 执行嵌套循环：外层遍历 $\text{List}_{\text{Forward}}$ 中的每一个 $K_1^*$，内层遍历 $K_2$ 的所有 $2^{56}$ 个可能取值 $j$。
2.  对于每一对 $(K_1^*, K_2=j)$，计算正向中间值 $B_{\text{fwd}}$。由于 $A=0$ 是已知条件，计算简化为 $B_{\text{fwd}} = D_j(A) = D_j(\mathbf{0})$。
3.  使用计算出的 $B_{\text{fwd}}$ 作为键，在 $\mathcal{H}_{\text{Backward}}$ 中查找是否存在匹配。
4.   如果找到匹配，即 $B_{\text{fwd}} = B_{\text{rev}}$，则取出哈希表中存储的 $K_3^*$.此时找到了一个候选密钥三元组 $\mathbf{K}_{\text{cand}} = (K_1^*, K_2=j, K_3^*)$。
5. 为了确认找到的密钥组合是正确的，使用另一个预先准备好的明文-密文对 $(P', C')$ 对 $\mathbf{K}_{\text{cand}}$ 进行加密验证。如果 $E_{K_3^*} ( D_j ( E_{K_1^*}(P') ) ) = C'$ 成立，则该 $\mathbf{K}_{\text{cand}}$ 即为正确的 3DES 密钥。

选择明文攻击需要$2^{56}$次查询，存储$2^{56}$个$(K_1,C^*)$对，计算$2^{112}$次DES运算，存储$2^{56}$个$B_{rev}$，总时间复杂度为$O(2^{112})$，空间复杂度为$O(2^{56})$。
